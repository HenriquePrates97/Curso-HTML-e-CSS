Configurando as características do iframe

Por padrão, a altura do iframe é de 150px e a largura é de 300px, mas para alterar esse padrão, podemos colocar um parametro height na tag para alterar a altura e um parametro width para alterar a largura.

Também é possível alterar as configurações de altura e largura utilizando CSS, que acabam se sobrepondo ao que está como parâmetro, basta apenas adicionar o nome da tag na CSS e fazer a alteração como qualquer outra tag em HTML, só lembrando, as CSS são soberanas, tudo que estiver como parâmetro na tag será desconsiderado.

Existe também um parâmetro chamado scrolling, que é utilizado para permitir ou não a rolagem da página que está dentro do iframe, caso essa página seja grande o suficiente para criar uma barra de rolagem, no iframe acima deixei o parametro em "no" (não em inglês), para bloquear a rolagem da página, as outra opções dessa parâmetro são o auto, onde o iframe decide se é necessário ou não a rolagem, e o yes, onde o iframe cria uma barra de rolagem mesmo que a página caiba dentro do iframe e não precise rolar.

O frameborder serve para indicar se o iframe terá borda visível ou não, sendo o valor 0 indicativo para sem borda e qualquer outro valor indicativo para borda visível. Também é possível estilizar a borda do iframe utilizando CSS

Para abrir dentro de um iframe uma página de um link da tag a, adicionamos ao iframe um name, definimos o target desse link, em vez de _blank, o name desse iframe no target.

Vamos agora falar sobre algumas inconveniências dos iframes, pois nem tudo são flores

Problemas na indexação de conteúdo pelos sites buscadores

Caso haja muitos iframes no seu site, pode ser que os bots de indexação dos sites de busca não entrem no conteúdo do iframe para avaliar a relevância desse conteúdo e assim decidir em que posição seu site aparecerá quando o usuário pesquisar por algum termo que remeta a seu site, fazendo com que ele não tenha um bom desempenho nas buscas e acaba ficando irrelevante. Por isso o mais aconselhado é utilizar iframes com moderação e não utilizar conteúdos mais importantes para o seu site dentro deles

Acessibilidade e usabilidade

Para pessoas que utilizam algum software para auxiliar na leitura da tela do computador o uso de iframe no site pode ser um inconveniente, pois esses softwares podem ter dificuldade em acessar as informações do iframe para fazer a leitura, assim como os própior navegadores que podem ter dificuldade de entender alguns comandos dados pelo usúario em relação ao iframe e acabar tornando a experiência não tão agradável assim

Segurança

Quando utilizamos um site de terceiros em nosso site atráves de um iframe, estamos dando passe livre para que esses sites façam parte do nosso conteúdo, e caso haja algum tipo de fraude ou golpe ocorrendo nesses site, ou até mesmo algum hacker invadiu este site e está utilizando ele para roubo de dados dos usuários, nosso site se torna mais uma porta de entrada para pessoas cairem nessas armadilhas e assim fazendo com que a segurança do nosso próprio site fique fragilizada.

E para conseguir deixar o site mais seguro, existem alguns parâmetros que podemos utilizar nos iframes para evitar que sites que estão sendo executados neles consigam efetuar algumas ações. Utilizamos o parametro sandbox="" para alterar as autorizações dos sites, como por exemplo rodas scripts, pegar informações de formulários, para negar todas as ações, escrevemos sandbox dentro das aspas, ficando assim: sandbox="sandbox". Dessa forma o site está um pouco mais seguro. Caso haja a necessidade de liberar algumas autorizações, há essa possibilidade, basta apertar control+espaço para verificar as opções disponiveis no parametro sandbox.

Outra forma de deixarmos o site mais seguro é evitar que os dados do usuário sejam coletados para elaboração de propagandas baseadas nas pesquisas desse usuário, para isso utilizamos o parametro referrerpolicy="" com a configuração no-referrer, assim, nenhum dado será utilizado pelo site que está sendo executado no iframe.
